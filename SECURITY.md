# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public issue. Instead, report it privately by opening a [GitHub Security Advisory](../../security/advisories/new) or contacting the maintainer directly.

Please include:

- A description of the vulnerability
- Steps to reproduce
- The affected version(s)
- Any potential impact

We will acknowledge your report within 48 hours and aim to provide a resolution or mitigation within 7 days.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |

## Scope

This policy covers the ZenMux Monitor application source code in this repository. It does not cover third-party dependencies or the ZenMux API service itself.
